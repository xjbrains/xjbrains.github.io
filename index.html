#简单粗暴本地git仓库配置

  • 切换到代码仓库
    xjswift:~ xj$ cd /Users/xj/Downloads/XJCodes/Repository/GitDemoProject
  • 初始化代码仓库
    xjswift:GitDemoProject xj$ git init –bare
  • 切换到本地代码文件夹
    xjswift:GitDemoProject xj$ cd /Users/xj/Downloads/XJCodes
  • clone代码Repository到本地文件夹
    xjswift:XJCodes xj$ git clone /Users/xj/Downloads/XJCodes/Repository/GitDemoProject
  • 进入clone下来的项目
    xjswift:XJCodes xj$ cd /Users/xj/Downloads/XJCodes/GitDemoProject
  • 配置自己项目的用户名和邮箱
    xjswift:GitDemoProject xj$ git config user.name “xj”
    xjswift:GitDemoProject xj$ git config user.email “xj@qq.com”
  • 添加.gitignore文件
    xjswift:GitDemoProject xj$ touch .gitignore
    xjswift:GitDemoProject xj$ open .gitignore
    xjswift:GitDemoProject xj$ git add .
    xjswift:GitDemoProject xj$ git commit -m “add swift gitignore file”
    xjswift:GitDemoProject xj$ git push .
    xjswift:GitDemoProject xj$ git lg
  • 353ac95 - (HEAD -> master) add swift gitignore file (24 seconds ago)
    xjswift:GitDemoProject xj$
  • 配置成功
  • 下面是swift .ignore文件内容复制粘贴就行

cocoapod世界上最快的安装方法

sudo /usr/bin/ruby -e “$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)”

brew update
brew install ruby

##CGD详解

  • 队列
    • dispatch_get_global_queue(0,0) 全局并发队列
    • dispatch_get_main_queue() 主队列
    • dispatch_queue_create(“xj”,DISPATCH_QUEUE_SERIAL) 自定义串行队列
    • dispatch_queue_create(“xj”,DISPATCH_QUEUE_CONCURRENT) 自定义异步队列（并发队列）
  • 串行队列和并发队列，同步任务和异步任务
    • 任务决定开不开线程
    • 队列决定开几条线程
      • 串行队列和并发队列都是队列，开的线程都是按队列（FIFO）不同的是串行队列只能带一条线程，然后将任务按先后顺序一一执行，而并发队列是有能力开多条线程能力的队列，说有能力开线程，就是说可以当串行队列的效果来用，也可当并行队列来用

  • dispatch_after 来完成任务的延迟执行

  • dispatch_once 的用法
    -dispatch_once表示一个程序只执行一次，他有个token用来标记执行了还是没有做，默认token的值是0，做了之后系统将其的值改为已所以在dispatch_once任务中传入的token为token的地址，并且token要是全局变量或静态变量（如果设置成别的没卵用）

  • dispatch_(a)sync任务和dispatchgroup(a)sync任务

  • dispatchbarrier(a)sync 任务用法

  • dispatch_apply的用法
